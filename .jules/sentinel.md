## 2024-05-30 - Fix Unix Domain Socket Creation TOCTOU Race Condition
**Vulnerability:** The Unix Domain Socket for IPC was created using default process umask permissions, making it vulnerable to unauthorized local connections. A naive fix using `set_permissions` immediately after `bind()` creates a Time-Of-Check to Time-Of-Use (TOCTOU) race condition window where an attacker can connect.
**Learning:** In Rust (and POSIX generally), creating a Unix Domain Socket with specific permissions securely and atomically requires manipulating the process `umask` *before* calling `bind()`, as `bind()` applies the current umask to the newly created socket file.
**Prevention:** Always temporarily set `umask(0o177)` before binding a UDS, and restore it immediately afterward. Ensure the `bind()` call is synchronous (not `.await`) to prevent async task cancellation from leaving the umask permanently modified.

## 2024-06-03 - Missing SO_PEERCRED Verification on IPC Socket
**Vulnerability:** The Unix Domain Socket created by the Rust sidecar used to receive execution requests from the Go Kernel lacked `SO_PEERCRED` validation. While `umask` was correctly set to prevent local TOCTOU attacks from creating unauthorized connections during startup, the underlying Tonic gRPC server natively accepts incoming stream connections blindly, creating an opening if another local user process ever circumvented the initial socket permissions.
**Learning:** In Rust/Tonic, a `UnixListenerStream` accepts all incoming connections by default without verifying peer credentials. To ensure true mutual capability checks over Unix Domain Sockets as detailed in `docs/IPC_SECURITY.md`, you must actively filter the connection stream.
**Prevention:** Always wrap `UnixListenerStream` with a `filter_map` (via `tokio_stream::StreamExt`) and extract the peer's credentials via `.peer_cred()` to strictly enforce UID requirements (e.g. `cred.uid() == expected_uid`) before passing the stream into Tonic's `serve_with_incoming()`.
