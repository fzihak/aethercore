## 2024-05-30 - Fix Unix Domain Socket Creation TOCTOU Race Condition
**Vulnerability:** The Unix Domain Socket for IPC was created using default process umask permissions, making it vulnerable to unauthorized local connections. A naive fix using `set_permissions` immediately after `bind()` creates a Time-Of-Check to Time-Of-Use (TOCTOU) race condition window where an attacker can connect.
**Learning:** In Rust (and POSIX generally), creating a Unix Domain Socket with specific permissions securely and atomically requires manipulating the process `umask` *before* calling `bind()`, as `bind()` applies the current umask to the newly created socket file.
**Prevention:** Always temporarily set `umask(0o177)` before binding a UDS, and restore it immediately afterward. Ensure the `bind()` call is synchronous (not `.await`) to prevent async task cancellation from leaving the umask permanently modified.

## 2026-06-28 - [CRITICAL] Fixed IPC Unix Domain Socket Security Vulnerabilities
**Vulnerability:** The Unix Domain Socket was subject to a TOCTOU race condition during creation where `umask` modifications could leak permanently if a panic occurred between setting and restoring the umask. Furthermore, the IPC endpoint lacked explicit peer credential verification (`SO_PEERCRED`), meaning other processes executing under the same user could theoretically connect to the IPC boundary if they bypassed filesystem permissions.
**Learning:** In Rust, temporary global state modifications (like `umask`) must be wrapped in an RAII guard (implementing `Drop`) to ensure panic-safety and prevent unintended leaks. For Unix Domain Sockets in Tonic/gRPC, filesystem permissions (like `0600`) offer a baseline defense, but verifying `SO_PEERCRED` strictly ensures that the connecting process matches the expected effective UID at the stream level.
**Prevention:** Always use RAII guards for `umask` modifications. Always use `StreamExt::filter_map` and `stream.peer_cred()` to validate the connecting process's UID before passing the UDS stream to a Tonic server.
